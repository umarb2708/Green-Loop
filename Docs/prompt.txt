Green Loop
Description: Smart plastic collection bin which collect the bins and provide points to the user who dispose it 
This project contains both Hardware and Software parts. 
There will be 4 chambers for a bins for disposing PET(bin1), HDPE(bin2), PP(bin3), Others(bin4) where the lid of this bins are controlled by servo motor. 

Hardware Details 
    1. ESP32 C3 Mini: Act as MCU
    2. 4x IR Sensor: To confirm plastic entry
    3. 4x SG90 servo: To open close lid
    4. 20KG load cell+HX711(I2C): To confirm the bottle disposal by checking weight chnage
    5. PCA9685 16 Channel Servo Motor Driver (I2C) : controll Servo 
    6. one push button (Start Session / Stop Session)
    7. Oled: For display

Hardware Working: 
Esp32 should use the Flash memory to store Wifi credential, IP address, bin id.
globals variables 
    disposal_started =0;
    tot_points =0; 
setup()
    1. call init_display()
    2. call Wifi_Setup()
    3. init_hw()
init_display();
    1. Initialize Serial Monitor
    2. Initialize OLED
        2.1: If OLED is initialized,  Display Project Name in OLED
        2.2: Else show error in Serial Monitor
Wifi_Setup(): (Inside setup function to run after reset or power on)
    1. Display Initiaizing Wifi in OLED & Serial Monitor
    2. Use the stored wifi credential to connect
    3. If no stored value or not connected to store wifi for 15 sec, then start config_mode with Display in OLED & Serial Monitor
    4. once new credential recieved or exit from config mode, then continue step 1
Config_Mode():(works only when functions is called)
    1. Change ESP32 to AP Mode with SSID: GreenLoop, Password: Green@123#
    2. Start a webserver with form of following field. Show the IP, AP SSID & Password in oled & Serial Monitor
        2.1: SSID: User can input the new WiFi SSID here
        2.2: Password: User can input the new WiFi Password here
        2.3: IP: this will the website or server IP address for API call 
        2.4: BinID: this is the bin_id value generate from creating a new bin from software side
    3. Store the values in Flash memory
    4. Change the ESP32 to STA mode and exit. 
init_hw()
    1. initialize PCA9685. If error, show error in OLED & Serial Monitor
    2. initialize HX711. If error, show error in OLED & Serial Monitor
    NB: If error, add a display format like "<hardware> is not initilaized, check wiring and do a reset to re initialize"

loop()
    1. read_bin_status()
    2. read_button_click()

read_bin_status(): (This works inside the loop function for every 5 sec and also works when called)
    1. check the total weight
    2. check the state of lid_status[bin_num] (closed or opened) according to servo position
    3. check the IR values. If output return High or Low
    4. for each bin_chamber, If IRVal[bin_num] == LOW &  lid_status[bin_num] = CLOSED, then bin_full[bin_num] =1 else bin_full[bin_num] =0
    5. Push weight, bin_full to 'bin_data'  table in the website database
    5. return weight, bin_status for each chambers like a 4 digit number.Eg: 1011 means bin[3] is not full(value at 3rd position is 0). 

read_button_click() (This works inside the loop function for every 5 sec and also works when called)
    1. check the button1 input
    2. if button1 == clicked & disposal_started ==0,  then make disposal_started =1 & call disposal function. Log it in Serial Monitor
    3. else  button1 == clicked & disposal_started ==1, then make disposal_started =0 & call rewards_earned function. Log it in Serial Monitor
    6. else do nothing and return
disposal(): 
    1. Initialize a variable tot_points == 0
    2. call read_button_click()
    3. wait for 10 sec Poll the disposal_data table from webserver where confimred == 0 && bin_id =BinID (from flash memory at startup or config mode)
    4. If an entry received from step2, then show "Disposing <Type of plastic>" in OLED & Serial Monitor , goto step 5 else step 2
    5. Trigger to open the lid of the particular chamber 
    6. Check IR Sensor status of that chamber for next 10 seconds
        6.1: If IR sensor detected, then goto step 7
        6.2: Else show error in OLED & Serial monitor & return to step 2
    7. Check the weight change in Load sensor
        7.1: If weight changed, then go to step 8 
        7.2: else Show error & go to step 2
    8. Update the points by adding this below reward points to tot_points
        PET = 10 points
        HDPE = 20 points
        PP = 30 points
        others = 5 points
    9. Update the row fetched at step 2 with confimred == 1
    10. Show the tot_points in OLED
    11. Go to step2. 
rewards_earned()
    1. Show "Disposal Completed" in OLED & Serial Monitor. 
    2. Generate a 6 character alphanumeric random value
    3. Call read_bin_status function for bin status  
    4. Push the points , alphanumeric value to the table 'rewards_data'.
    5. Shows the points & the alphanumeric value in OLED screen

MySQL tables 
    1. users: This contains the user login info. it contains following coloumn
      a. id: Auto increment primary key 
      b. Name: Name of the user 
      c. username: username for the user. This should be unique 
      d. password: password for the user
      e. is_admin: tinyint(1) value informs that the current use is admin or not. 1: Admin 0: Normal user
      d. rewards_collected: This is the total reward points collected by the user. Applicable for Normal user only. 
    2. product_data
      a. id: Auto increment primary key
      b. qr_id: this a random 10 character alphanumeric value when adding product by admin. This is used by the hardware while scanning plastic
      c. Manufacturer: This is the Manufacturer name for that bottle
      d. type: This shows the type of plastic 
    3. bin_data
      a. id: Auto increment primary key
      b. loaction: This will hold the location of the smart bin is placed
      c. Current Status:  4 digit value informs whether the bin is full or not with 1 or 0 at the bin position. Eg: 0001 show bin4 is full. 
      d. weight: current weight of bin from last entry
    4. rewards_data
      a. id: Auto increment primary key
      b. unique_code: this is the alphanumeric code generated at step 11.2 of hardware Working
      c. points: this is the total points calculated at at step 11.1 of hardware Working
      d. collected: This will shows that the reward is already collected or not. default to 0. 
    5. disposal_data
      a. id: id: Auto increment primary key
      b. bin_id: This is the in id where disposal is happening. 
      c. type: type of plastic
      d. confirmed: This will update the disposal entry is processed by hardware or not. default to 0.

API Details
    1. upload_data: used to upload points , alphanumeric value to the table 'rewards_data' & weight, bin_full to 'bin_data'  table along with user id. uses below JSON
        {
            "Points": points, 
            "unique_code": alphanumeric value
            "bin_weight": weight
            "bin_status": bin_full for each chambers
            "bin_id": sends the bin ID from Flash memory
        }
